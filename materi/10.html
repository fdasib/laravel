Factory & Faker

Intro

  • akan kita gunakan ketika kita ingin membuat banyak data sekaligus secara otomatis untuk nantinya kita jadikan sebagai data didalam seeder kita
  • kalo seeder kita menuliskan datanya manual 1 per 1 didalam seedernya
  • kita menggunakan sebuah library dinamakan faker : untuk membuat data kita terisi otomatis dengan data palsu / data dumi yang masuk akal secara random
  • jadi kita tidak perlu memikirkan data apa yang akan kita masukkan kedalam seedernya

kita coba fitur keren factory

  • factory == pabrik
  • pabrik pembuat data
  • datanya data palsu
  • ketika kita masuk di model Post.php
  • itu ada script use HasFactory
  • jadi ini sebenarnya secara default kita sudah bisa bikin factorynya
  • sama seperti dimodel User
  • kalo kita tekan ctrl + click
  • nanti dia akan masuk ke sebuah trait yang namanya factory
  • dimana disini kita bisa bikin factorynya
  • kita masuk ke seeder DatabaseSeeder.php
  • kita lihat dokumentasinya

factory

  • pada saat pengujian, kita mungkin saja / pasti butuh menambahkan data baru ke dalam database kita sebelum menjalankan test
  • dari pada kita secara manual mengetikkan datanya setiap columnnya
  • laravel memungkinkan kita untuk mendefinisikan sebuah factory ke dalam model kita
  • untuk melihat cara kerja nya lihat aja database\factories\UserFactory.php
  • isinya adaah ini:
  • isinya sebuah class yang terhubung ke class Factory
  • dimana didalamnya ada sebuah method definition, untuk bikin data user secara random
  • jadi kita bikin dengan skema yang sama dengan skema yang kita bikin dimodel
  • contoh :
  • nama : dia memanggil sebauh method yang namanya faker name
    • ini artinya generate nama palsu pakai library faker
  • email
    • generate email palsu yang sudah safeEmail(format nya sudah benar) yang dia uniq pakai library faker
  • kita bisa bikin factory kita sendiri

kita benerin agar tidak pakai user

  • ganti di web.php nya author
  • dipost.blade.php kita manggil nya tetap user, nah ini konteks nya tidak sesuai kalo misal kita mau ubah user ini menjadi author
  • kita cari model Post.php
  • nah dimodel ini kita memanggil nya user, nah saat dihalaman view post nya memanggil user itu memanggil method ini
  • nah kalo kita ganti jadi author ini akan error karena laravelnya akan mengecek ada tidak ditable post itu field yang namanya author_id sebagai foreign key nya
  • nah padahal gak ada adanya juga user_id
  • nah kalo misal kita mau mengganti jadi author kita tambahkan parameternya user_id di file model Post.php
  • jadi kita mau ganti user_id aliasnya / nama lainnya jadi author
  • kalo kita balik / mengecek hasilnya error semua karena kita menuliskan user
  • kita benerin
  • kita ke view posts.blad.php
  • kita edit juga view dihalaman post.blade.php
