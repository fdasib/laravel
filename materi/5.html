Apa itu model

  • masuk ke dalam menu Eloquent Model
  • Eloquent : fitur yang ada didalam laravel yang merupakan ORM(Object Relasional Model)
  • masuk ke getting started
  • didalam laravel ada sesuatu yang disebut Eloquent
  • dia adalah sebuah object-relasional-mapper(ORM) yang membuat kita itu dengan mudah berinteraksi dengan database
  • dengan menggunakan Eloquent ini setiap table didalam database itu punya sebuah model bentuk nya class, agar kita bisa berinteraksi
  • kita bisa dengan mudah memanipulasi table dengan model, mulai dari insert, update, delete kedalam table
  • intinya dia sebuah kelas untuk mengelola table didalam database

Membuat model

  • kita menggunakan perintah artisan
    • php artisan make:model {nama model}
  • kita juga bisa membuat migrationnya
    • jadi migration ini untuk menuliskan skema database dan juga nanti akan mengenerate akan menjadi table didalam database
  • terminal / extension dari visual studio code
  • kalo lewat extension
    1. ctl + shift + p
    2. ketikkan artisan pilih Model
    3. nama model
    4. apakah akan terhubung ke migrasi database : pilih no dulu
    5. migration no lagi
    6. Factory no lagi
    7. otomatis bikin controller : no
    8. kalo sudah kita lihat di file explorer kita Models/Post.php
  • kalo manual dari terminal
    1. masuk terminal
    2. pastikan posisi berada di folder projectnya
    3. ketikkan php artisan make:model {nama modelnya}
    4. lihat di file App/Models/Post.php
  • otomatis akan dibuat kan script ini karena untuk terhubung dengan database
  • tapi karena kita belum butuh maka bisa kita hapus dulu
  • kita sudah punya class post untuk menyimpan data dari postingan kita
  • sehingga diroutes kita ini kita gak perlu nulisin lagi data potingannya di routes/web.php
  • kita cut data blog_post dari routes/web.php ke App/Models/Post.php
  • sebelum dipaste kita bikin property statis dulu difile App/Models/post.php
  • karena kita hanya pengen bisa diakses oleh class ini saja jadi akses modifier nya private

Collection

  • kota bisa mengubah data yang kita ambil menjadi sesuatu yang dinamakan colection
  • Collection : pembungkus sebuah array, yang akan membuat array menjadi lebih sakti
  • lihat didokumentasi
  • didalam laravel menyediakan sebuah collection, yang merupakan pembungkus dari data array
  • dengan collection memungkinkan kita bisa menjalankan banyak function
  • salah satunya map
  • caranya bungkus dengan collect
  • kalo dijalankan tidak ada perubahan / error
  • untuk mengambil data pertamanya
  • makanya pakai firstWhere

Controller

  • ketik di dokumentasi controller
  • dari pada kita mendefinisikan semua logika program kita sebagai closure didalam route file kita, kita bisa merapikan ini menggunaan sebuah class controller
    • closure : anonymous function
  • controller bisa menggabungkan request serupa untuk menangani logic nya kedalam sebuah class
  • cara membuatnya kita bisa sebuah controller yang tersimpan dinamespace
  • terus bikin class didalamnya, class nya biasanya ada namanya diakhiri dengan controller yang mewarisi semua prilaku dari kelas controller
  • didalamnya kita punya sebuah method untuk menangani sebuah logicnya
  • jadi kita punya 2 method
    1. Menangani sebuah postingan
    2. menangani satu buat postingan
  • kalo methodnya sudah jadi siroutenya tidak lagi punya clossure tapi kita tulis :
  • kita tulis requestnya(rutenya mau kemana), [] : nama controllernya apa, 'show' : nama methodnya apa
  • cara bikinya adalah kita bisa pakai artisan
  • cek di file App\Http\Controllers\PostController.php
  • kita biarkan dulu use nya, karena untuk menangani request kalo punya form, data diURL
  • didalamnya nanti kita mau bikin 2 buah method
  • kita ambil datanya dari web.php
  • kita hubungkan dengan App\Models\Post.php
  • kita perbaiki di route/web.php kita, kita sudah tidak pakai closeure kita
  • jangan lupa kita includekan
  • buat method berikutnya : show() : untuk menampilkan detail 1 postya
